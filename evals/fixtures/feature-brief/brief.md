# Feature Brief: Team Workspaces

## Overview
Allow users to create shared workspaces where team members can collaborate on
projects. Each workspace has its own settings, members, and project list.

## Target users
- **Workspace admin** — creates the workspace, manages members
- **Team member** — joins workspaces, works on projects within them

## Requirements
- Admins can create a workspace with a name and description
- Admins can invite members by email
- Members can view all projects in their workspace
- Members can be removed by admins
- Each workspace has a settings page (name, description, default timezone)
- Workspace data is isolated — members only see their own workspaces

## Success metric
- 60% of new teams create a workspace within first week

## Out of scope
- Billing per workspace (handled separately)
- Cross-workspace search
